# Feature brief guidance

Files in this directory describe high-level product vision and feature direction. Keep them durable and concise; do not use them as detailed live specifications of current functionality.

Put exact behavior, implementation scope, acceptance criteria, edge cases, and testing expectations in the executable Beads that need them. When one detailed contract is shared by enough Beads that copying it would cause duplication or drift, record it once in the relevant roadmap phase and have those Beads reference it.
